# Contribution Guidelines

Thank you for your interest in contributing to the WiFi-Coin Wallet! We appreciate your efforts to help us improve the project. Please follow these guidelines to ensure a smooth and positive experience for everyone involved.

## Getting Started

1. **Fork the repository**: Use the GitHub interface to create a personal copy of the repository.
2. **Clone your fork**: Clone your forked repository to your local machine.
3. **Create a new branch**: Before making changes, create a new branch for your feature or bug fix.

   ```bash
   git checkout -b my-feature-branch
   ```

## Making Changes

- Ensure your code adheres to the [Code Standards](#code-standards) outlined below.
- Write clear, concise commit messages. Use the present tense, e.g., "Add feature" instead of "Added feature."
- Keep your commits small and focused on a single change.

## Code Standards

- Follow the [JavaScript Standard Style](https://standardjs.com/) for JavaScript code.
- Use consistent indentation (2 spaces for JavaScript).
- Write comments where necessary to explain complex logic.

## Testing Your Changes

- Ensure all tests pass before committing your changes.
- Consider adding new tests for new features or bug fixes.

## Submitting Your Changes

1. **Push your changes**: Push your changes to your remote repository.

   ```bash
   git push origin my-feature-branch
   ```

2. **Create a Pull Request**: Navigate to the original repository and create a pull request. Provide a clear description of your changes and reference any related issues.

## Code Review

- Be open to feedback from maintainers and other contributors.
- Make any requested changes promptly to ensure your contribution can be merged.
